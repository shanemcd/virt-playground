# kubevirt-ui/kubevirt-plugin

The OpenShift console plugin that adds the Virtualization section to the sidebar.

- Repo: [kubevirt-ui/kubevirt-plugin](https://github.com/kubevirt-ui/kubevirt-plugin)
- Local clone: `~/github/kubevirt/kubevirt-plugin`
- Deployed version: v4.21.0
- Namespace: `console-plugin-kubevirt`

## What it is

A static React frontend served by nginx. No backend logic, no controllers, no CRDs. The plugin talks to KubeVirt's API through the OpenShift console's built-in proxy.

It adds the Virtualization section to the console sidebar with pages for VMs, Templates, Instance Types, Migrations, and a dashboard.

## How OpenShift console plugins work

OpenShift console supports dynamic plugins via the `ConsolePlugin` CRD. The mechanism:

1. A `ConsolePlugin` CR registers the plugin with the console, pointing to a Service that serves the JS bundle
2. The plugin is listed in `consoles.operator.openshift.io/cluster.spec.plugins`
3. The console operator restarts the console pod to load the new plugin
4. The console fetches the JS bundle from the plugin's Service over TLS and injects it into the UI
5. The plugin's React code adds nav items, pages, and dashboard widgets

The TLS cert for the plugin Service is auto-generated by OpenShift's service-ca via the annotation `service.alpha.openshift.io/serving-cert-secret-name`.

## Components

| Resource | What it does |
|----------|-------------|
| `Deployment` | Runs nginx serving the JS bundle on port 9443 over TLS |
| `Service` | ClusterIP service fronting the deployment, triggers cert generation |
| `ConsolePlugin` CR | Registers the plugin with the console (name, service, port, base path) |
| `ConfigMap` (nginx-conf) | nginx configuration for TLS and health endpoint |

## Gotcha: ConsolePlugin name must match plugin-manifest.json

The `ConsolePlugin` CR name must exactly match the `name` field in the plugin's `plugin-manifest.json` (baked into the image). For the v4.21.0 image, the manifest says `"name": "kubevirt-plugin"`, so the ConsolePlugin CR must be named `kubevirt-plugin`.

The upstream repo's template (`oc-manifest.yaml`) uses `console-plugin-kubevirt` as the name, which doesn't match. It also references a broken image URL and uses the old `v1alpha1` ConsolePlugin API. Don't use it as-is.

## Security context

The plugin pod runs as a standard unprivileged nginx container. No host access, no special capabilities. It only serves static files.

## Relationship to other projects

In the productized stack (OpenShift Virtualization), HCO deploys this plugin automatically. With upstream KubeVirt, it's a separate manual install. The plugin lives in the `kubevirt-ui` GitHub org, not under `kubevirt` or `openshift`.

The Forklift console plugin (`kubev2v/forklift-console-plugin`) is a separate plugin that adds the Migration section. It follows the same pattern but comes from a different repo and adds different pages.
